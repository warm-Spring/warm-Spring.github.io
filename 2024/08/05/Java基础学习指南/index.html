Java基础

面向对象和面向过程的区别

面向过程是把解决问题的过程拆成一个个方法，通过一个个方法的执行解决问题。

面向对象是把解决问题的方式抽象成对象，利用对象执行方法解决问题。

面向对象三大特性

  1. 封装：封装是把一个对象的状态信息隐藏在对象内部，不允许外部直接访问对象内部信息，但是会提供一些方法可供外界访问。（安全性）

  2. 继承：一个子类继承父类，子类可以继承父类的所有方法和属性。就是在一个已存在类的基础上建立新类的技术，新的类可以增加新的定义和新的功能。（可重用性和扩展性）

  3. 多态：表示一个对象可以有多种状态，具体的表现是父类的引用指向子类的实例。
    声明一个对象可以是它的父类或接口，但在运行时执行它的方法是根据对象的实际类型来执行的。

接口和抽象类区别

相同点：

  1. 接口和抽象类不能直接被实例化，要通过实现或者继承才能实例化。
  2. 接口和抽象类都可以包含抽象方法。

不同点：

  1. 接口强调的是类功能的实现，对行为的约束，如Cook接口，实现类实现Cook类里的功能。抽象类强调的是类的从属关系，如Animals抽象类，子类为cat,dog
  2. 一个类只能继承一个抽象类，可以实现多个接口。
  3. 抽象类可以有构造函数，接口没有
  4. 抽象类可以有抽象方法和非抽象方法，接口只有抽象方法，后来jdk1.8引入default可以非抽象方法。

重写和重载

  1. 重写是子类对父类的方法重写。方法名、参数列表、返回值(子类中方法的返回值可以是父类中方法返回值的子类)都相同。方法体可以修改。子类函数的访问修饰权限不能少于父类的。

  2. 重载是同一类中，不同方法的重载。方法名是相同的，参数列表不同，返回值可以不同（无法以返回型别作为重载函数的区分标准）。

Java异常机制

异常的顶层是Throwable，有两个子类Exception,Error。

Exception又分为两种，受检查异常(Checked Exception,必须处理)和不受检查异常(Uncheck Exception，可以不处理)。

不受检查异常(Uncheck Exception)也就是运行时异常，如：NullPointerException空指针异常，ArrayIndexOutOfBoundsException数组越界异常，ClassCastException类型转换错误。

受检查异常是除了运行时异常之外的异常，如IOException, ClassNotFoundException, SQLException, FileNotFoundException。

Error是程序无法处理的错误，如OOM内存溢出、StackOverFlowError栈溢出。

  • 当 try 语句和 finally 语句中都有 return 语句时，try 语句块中的 return 语句会被忽略。这是因为 try 语句中的 return 返回值会先被暂存在一个本地变量中，当执行到 finally 语句中的 return 之后，这个本地变量的值就变为了 finally 语句中的 return 返回值。

反射

  • 概念：通过反射可以在运行时动态的获取任意一个类的方法和属性，并且可以调用。也就是在程序运行时动态加载类并获取类的详细信息，从而操作类的方法和属性。
  • 本质：JVM得到class对象之后，再通过class对象进行反编译，从而获取对象的各种信息。
  • 应用场景：注解、Spring\MyBatis框架

ArrayList扩容机制

使用无参构造时初始容量为0；当添加第一个元素时，默认容量为10；当添加的元素超过容量时，会扩容为原来的1.5倍。

扩容策略：ArrayList在扩容时，会创建一个新的容量更大的数组，并将原有元素复制到新数组中。

HashMap集合

底层结构

jdk1.7之前是数组+链表的结构，jdk1.8之后是Entry数组+链表/红黑树。当链表的节点个数达到8时，会将链表转为红黑树；当红黑树的节点个数小于等于6时，又会将红黑树转为链表（在红黑树上增加了双向链表，是为了更好的相互转换）。在转换为红黑树之前会先判断当前数组的容量是否超过64，若未超过，则会优先进行数组扩容。

（阈值为8，原因为开发者经过实践，节点数超过8的概率很小，几百万分之一。若发生则说明往后冲突概率会变大。）

get()方法
  1. 首先，需要计算键的哈希值，并通过哈希值计算出在数组中的索引位置。
  2. 如果该位置上的元素为空，说明没有找到对应的键值对，直接返回null。
  3. 如果该位置上的元素不为空，遍历该位置上的元素，如果找到了与当前键相等的键值对，那么返回该键值对的
    值，否则返回null。
put()方法
  1. 首先，put方法会计算键的哈希值（通过调用hash方法），并通过哈希值计算出在数组中的索引位置。
  2. 如果该位置上的元素次空，那么直接将键值对存储在该位置上。
  3. 如果该位置上的元素不为空，那么遍历该位置上的元素，如果找到了与当前键相等的键值对，那么将该键值对
    的值更新为当前值，并返回旧值。
  4. 如果该位置上的元素不为空，但没有与当前键相等的键值对，那么将键值对插入到链表或红黑树中（如果该位
    置上的元素数量超过了一个阈值，就会将链表转化为红黑树来提高效率）。
  5. 如果插入成功，返回被替换的值；如果插入失败，返回null。
  6. 插入成功后，如果需要扩容，那么就进行一次扩容操作。
remove()方法
  1. 首先，remove方法会计算键的哈希值，并通过哈希值计算出在数组中的索引位置。
  2. 如果该位置上的元素为空，说明没有找到对应的键值对，直接返回null。
  3. 如果该位置上的元素不空，检查是否与当前键相等，如果相等，那么将该键值对删除，并返回该键值对的
    值。
    4.如果该位置上的元素不为空，但也与当前键不相等，那么就需要在链表或红黑树中继续查找。
    5.遍历链表或者红黑树，查找与当前键相等的键值对，找到则将该键值对删除，并返回该键值对的值，否则返回
    null.
如何定位key(通过哈希值计算出数组中的索引位置)

先通过（table.length - 1）& （key.hashCode ^（key.hashCode >>> 16））定位到key位于哪个table中，然后再通过 key.equals（rowKey）来判断两个key是否相同，综上，是先通过hashCode和来定位KEY的。

  • 为什么要重写hashcode 和equal方法？
    所以，在使用HashMap的时候，尽量用String和Enum等已经实现过hashCode和equals方法的官方库类，如果一定要自己的类，就一定要实现hashCode和equals方法。
为什么（table.length - 1）& （key.hashCode ^（key.hashCode >>> 16））

1.为什么是用&而不是用%：因为&是基于内存的二进制直接运算，比转成十进制的取模快的多。以下运算等价：X % 2^n = X & （2^n - 1）。这也是hashMap每次扩容都要到2^n的原因之一。
2. 为什么用key.hash^（key.hash >> 16）而不是用key.hash：这是因为增加了扰动计算，使得hash分布的尽可能均匀。因为hashCode是int类型，虽然能映射40亿左右的空间，但是，HashMap的table.lengthequals毕竟不可能有那么大，所以为了使hash%table.length之后，分布的尽可能均匀，就需要对实例的hashCode的值进行扰动，说白了，就是将hashCode的高16和低16位，进行异或，使得hashCode的值更加分散一点。

HashMap扩容机制

HashMap刚创建时，若没有指定大小，则默认大小为16。之后每次扩容，扩容为原来的2倍。若指定大小，则扩充为该大小的2的幂次方。负载因子是0.75（泊松分布）.

当达到负载因子时：

  1. 如果某桶节点没有形成链表，则直接rehash到其他桶中
  2. 如果桶中形成链表，则将链表重新链接
  3. 如果桶中的链表已经形成红黑树，但是链表中的元素个数小于6，则进行取消树化的操作

HashMap在并发下不安全的原因

  1. 在put元素时，两个线程同时put，则会发生一个覆盖另一个元素情况。
  2. 在put元素时，上述情况下，计算size值就会减少
  3. 当既有get操作，又有扩容操作的时候，有可能数据刚好被扩容换了桶，导致get不到数据。
  4. 在jdk1.7之前扩容时是头插法，1.8之后是尾插法。头插法会导致死循环。

HashMap,HashTable,ConcurrentHashMap区别

  1. HashMap是线程不安全的，另外两个线程安全
  2. HashMap是Key可以为null(只能一个),多个value可以为null；而另外两个不可以。（因为可能造成二义性，当一个线程插入，另外一个线程删除，当该线程获取key的时候，取出来null。但是不知道该null是指null还是不存在。）
  3. 底层数据结构不同
  4. 初始容量大小和每次扩充容量大小的不同： ① 创建时如果不指定容量初始值，Hashtable 默认的初始大小为 11，之后每次扩充，容量变为原来的 2n+1。HashMap 默认的初始化大小为 16。之后每次扩充，容量变为原来的 2 倍。② 创建时如果给定了容量初始值，那么 Hashtable 会直接使用你给定的大小，而 HashMap 会将其扩充为 2 的幂次方大小

ConcurrentHashMap的底层数据结构

JDK1.8之前是段segment水数组+链表的形式。当数据修改时，会锁住整个段数组。JDK1.8之后时Node数组+链表/红黑树的形式。当数据修改时，只会锁住一个Node节点。锁粒度更细。通过Node+CAS+synchronized来保证并发安全的。

红黑树

红黑树是自平衡二叉搜索树。（与二叉树相比，二叉树在顺序或逆序情况下退化为了链表，由O(logn)退化为O(n);与AVL相比，AVL在插入和删除时，需要维护AVL很耗时）红黑树适合频繁插入和删除，更加稳定。

性质：

  • 根叶全黑
  • 无相邻的红色节点
  • 节点到任意叶节点路径，所含黑色节点数量相同。
