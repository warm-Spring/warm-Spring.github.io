Jdk新特性

Jdk8新特性

  1. lamada表达式

(a,b)->{return a-b}

  1. stream流
    stream().map().filter().mapToInt().toCollect();

  2. 函数式编程
    Function(): a func(T b);
    Consume(): void func(T b);
    Supplier(): a func();
    Runnable(): void func();
    Predicate(): boolean test(T t);

  3. 方法引用

LambdaInterface t = LambdaClass::staticF;

  1. Optional防止空值

  2. default

  3. 新的日期API

java.time.*;LocalDate

Jdk11新特性

  1. var 本地变量类型推断

var value = new MyObject ();

  1. try-with-resource增强
    允许使用（）外面的变量

    工作原理：在 try-with-resources 语句中声明的资源会在 try 语句块结束时自动调用 close() 方法关闭资源。这一过程由 Java 虚拟机（JVM）自动管理，即使在 try 语句块中发生了异常也不会影响资源的关闭。这里的 scanner,writer 必须实现 AutoCloseable 接口，该接口定义了一个 close() 方法，用于关闭资源。

  2. G1成为默认垃圾回收器（jdk9）

  3. 引入ZGC垃圾回收器

Jdk17新特性

  1. switch yield返回

  2. ‘’’ ‘’’文本块可转义

  1. Record字段类

  2. sealed permits封闭类

  3. instance of模式匹配

  4. switch模式匹配

Jdk21新特性

  1. 虚拟线程
  2. 分代ZGC

jdk垃圾回收器默认

在 Java 8 的时候，默认垃圾回收器是 Parallel Scavenge（新生代）+Parallel Old（老年代）。到了 Java 9, CMS 垃圾回收器被废弃了，G1（Garbage-First Garbage Collector） 成为了默认垃圾回收器。
